第一单元：初识物联网

1. 核心发展史与概念

  • 1982年：卡内基梅隆大学的可乐售卖机(Only)，最早的物联网雏形。
  • 1991年：剑桥大学特洛伊咖啡壶（摄像头+网络）。
  • 1995年：比尔·盖茨《未来之路》提出构想。
  • 1999年：麻省理工首次提出“物联网”概念，并与 RFID（射频识别） 结合。

🔥 2. 物联网三层架构 (全书灵魂，必考！)

架构层级核心功能关键技术/设备
应用层 (大脑)对数据进行计算、处理、挖掘，转化为具体应用服务云计算、大数据、人工智能、各种软件平台(Web/App)
网络层 (神经)将感知层数据准确高效地传输到处理中心Wi-Fi, 蓝牙, ZigBee, LoRa, NB-IoT, 5G, IPv6, MQTT协议
感知层 (五官)感知、识别和采集物理世界中的各种信息(声光温等)传感器, RFID, 二维码, GPS, 摄像头

三大特征：全面感知 可靠传递 智能处理

第二单元：感知层初体验 (含代码)

1. 二维码与 RFID 识别技术

二维码：黑白方块二进制(黑1白0)。优势是信息量大、容错纠错能力强(抗损)。安全措施是动态二维码(防截图)。

RFID(射频识别)：利用无线电波双向通信。分为标签和读写器。优势是批量快速读取、防污耐高温。缺点是金属/液体环境易失效。

2. 红外传感器与算法优化

对射式：两端对齐，直射光，抗干扰强。漫反射式：单侧安装，靠物体反光，易受环境光干扰。

第三单元：网络层技术与MQTT协议 (全书核心)

1. 通信技术深度分类

  • 近距离通信：
    • Wi-Fi：局域网高速传输，工作在 2.4GHz 和 5GHz 免授权(ISM)频段。
    • 蓝牙 (Bluetooth)：极低功耗，距离约10米，常用于设备配对（一主多从）。
    • ZigBee：低功耗，自组网能力强，适合智能家居传感器网络。
  • 远距离通信：
    • LoRa & NB-IoT：低功耗广域网（LPWAN），速率低但覆盖极广，特别适合偏远地区农业大棚/环境监测。

🔥 2. MQTT 协议原理与代码剖析 (考点密集区)

特性：轻量级、低带宽，专为物联网设计。

通信机制：发布——订阅 (Pub-Sub) 模式。包含三个角色：发布者、订阅者、服务器代理(Broker)。双方不直接建立连接，而是依靠 Topic(主题) 进行数据路由。

QoS(服务质量)：QoS 0(最多一次，发完不管)、QoS 1(至少一次，保证到达可能重复)、QoS 2(只有一次，最可靠但耗资源)。

💻 代码实操：Wi-Fi 连接与 MQTT 客户端 (教材P63/69/70)

from unihiker_k10 import wifi, mqttclient
# 1. 必备第一步：连接 Wi-Fi 局域网
wifi.connect(ssid="School-WiFi", psd="123456", timeout=50000)
if wifi.status():
    print(wifi.info()) # 返回IP地址等信息

# 2. 建立 MQTT 服务器连接 (常用SIoT本地服务器，端口1883)
mqttclient.connect(server="192.168.43.86", port=1883, user="siot", psd="dfrobot")

# 3. 发布者 (Publisher) 发送数据：向特定 Topic 发送 Content
mqttclient.publish(topic="siot/msg", content="hello")

# 4. 订阅者 (Subscriber) 接收数据：必须定义回调函数
def on_message_recv(msg):
    print("收到消息:", msg)
# 订阅该主题，绑定回调函数
mqttclient.subscribe(topic="siot/msg", callback=on_message_recv)

第四单元：智能家居系统实践 (硬核控制逻辑)

1. 温度传感器与边缘计算

边缘计算原理：传感器采集的数据不上传云端，直接由本地控制中心（如行空板主控）完成判断与响应。优势是：无网络延迟、断网不断联、保护隐私数据。

    else:

2. 远程开关窗：舵机控制与异常防崩逻辑

智能家居控制流：感知 ➔ 采集 ➔ 传输 ➔ 分析 ➔ 反馈 ➔ 控制(执行器)。舵机(Servo)即为执行器。

💻 代码实操：安全的舵机控制指令 (教材P94)

from unihiker_k10 import servo
s1 = servo(1) # 初始化接入引脚1的舵机
# 假设从 MQTT 接收到了指令变量 msg
try:
    angle = int(msg) # 尝试将网络传来的字符串强转为整数
    # 【考试重点】保护硬件：无论传什么值，强制约束在 0 到 180 度之间
    angle = max(0, min(180, angle)) 
    s1.angle(angle) # 控制舵机转动开窗
except Exception as e:
    # 如果 msg 传来了字母 "open"，int("open")会报错，被这里拦截，防止程序崩溃
    print("收到非法指令，忽略操作", e)

第六单元：物联网安全与发展

1. 各层面的典型安全威胁 (必考概念)

  • 感知层：物理破坏、电磁波信号干扰（导致传感器发送错误脉冲）、克隆复制RFID门禁卡。
  • 网络层：
    • 中间人攻击 (MitM)：黑客悄悄潜伏在通信两端之间，截获或篡改通信数据。
    • Wi-Fi 钓鱼：在公共场所伪造免费Wi-Fi热点，盗取用户账号密码。
    • DDoS 攻击 (分布式拒绝服务)：控制海量带有漏洞的物联网设备（僵尸网络），同时向目标发送巨量无用请求，导致服务器瘫痪。
  • 应用层：API（应用程序接口）缺少权限验证导致越权操作、数据库密码被暴力破解。

2. 身份认证与自主可控战略

身份认证方式：1. 密码认证（需避免弱口令）；2. 生物特征认证（指纹、人脸、虹膜，基于人体生物特征的唯一性和不可复制性）；3. API令牌认证（程序间通信）。

自主可控（国家命脉）：
物联网设备高度依赖芯片和底层操作系统。分为高端芯片（如AI计算芯片，计算力强，极易被国外“卡脖子”）和低端芯片（如温湿度传感器）。
《中华人民共和国网络安全法》要求，关键信息基础设施必须实现自主可控，拒绝底层依赖，确保国家数据安全不受外部断供或恶意后门的威胁。
