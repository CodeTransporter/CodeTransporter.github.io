3.阿帕网（ARPANET）互联网雏形，选择必考完整知识点

核心意义：本次通信虽仅完成部分数据传输，但标志着互联网雏形正式诞生，证明跨地域计算机可通过网络实现数据交换，打破空间通信壁垒。

三、IP地址核心知识点（全书重难点，选择填空简答全覆盖，全段逐词标注）

1.IP地址设备网络唯一编号，必考名词定义：IP地址是互联网中每一台主机的唯一逻辑标识，相当于设备在网络中的专属“住址”，是网络识别、定位设备的核心依据。☆收藏

2.IP地址完整结构：由网络号 + 主机号组成要素必考填空两部分组成，二者缺一不可☆收藏

网络号：标识设备所属的网络区域（类比邮政编码，确定大范围区域）

主机号：标识该网络区域内的具体设备（类比门牌号，确定精准设备）

3.IPv4 完整概念

组成：采用32位二进制数必考数字填空编制，分为4组8位二进制数，转换为十进制展示（例：192.168.1.1）☆收藏

特点：地址总量固定、数量有限，无法适配当下海量物联网设备接入需求，目前已面临地址枯竭问题

4.IPv6 完整概念

组成：采用128位二进制数和IPv4对比必背数值编制☆收藏

特点：地址空间呈指数级扩大，可满足全球万物互联的设备接入需求，具备安全性高、传输稳定、适配性强的优势，是未来互联网的核心地址协议

5.IPv4与IPv6核心对比（必背对比考点）

IPv4：32位二进制、地址有限、安全性较弱、传统互联网通用

IPv6：128位二进制、地址无限、安全性更强、适配物联网/智能设备

三、HTML核心标签大全（定义+属性+用法·必考）

1. 网页基础骨架标签（层级固定必背）

<html>：网页根标签，包裹网页所有代码内容，是整个网页最外层结构

<head>：网页头部标签，存放网页元数据、标题、样式配置，内容不会在页面可视化展示

<title>：网页标题标签，定义浏览器标题栏显示的文字名称

<body>：网页主体标签，所有可视化的文本、图片、按钮等内容均存放于此

💡小提醒：html包裹head+body，标签嵌套不能写反！

2. 文本排版标签

<h1>~<h6>六级标题，h1最大权重最高：六级标题标签，h1字号最大、权重最高，h6最小，用于页面标题层级区分☆收藏

<p>：段落标签，自动实现段落换行、预留间距，用于正文文本排版

3. 媒体与超链接标签（核心属性必考）

<img>图片单标签，无闭合：图片单标签，用于插入网页图片

核心属性：src（必填，图片路径）、alt（图片失效替代文字）、width/height（尺寸px）☆收藏

<a>超链接双标签，href填跳转地址：超链接双标签，实现页面跳转

核心属性：href（必填，跳转目标网址/页面路径），格式：<a href="目标地址">显示文字</a>☆收藏

4. 表格标签

<table>：表格外框标签，定义整体表格

<tr>：表格行标签，定义表格中的一行内容

<td>：单元格标签，定义表格中单个单元格，存放具体内容

四、HTML标签核心规则（必背）

1.双标签：成对出现，有开始标签和结束标签（例：<p></p>、<body></body>）

2.单标签：无结束标签，独立使用（例：<img>、<br>换行标签）

3.固定层级关系：<html>包含<head>和<body>，<head>与<body>为并列关系，不可嵌套错乱☆收藏

三、TCP/IP四层协议模型（必考重难点·逐层解析）

1. 应用层：面向用户的功能应用层，承载各类网络服务，对应核心协议：

  • HTTP（网页浏览协议）访问网页专用通信协议☆收藏
  • SMTP（发送邮件协议）负责向外发送电子邮件☆收藏
  • POP3/IMAP（接收邮件协议）负责收取电子邮件☆收藏

2. 传输层：负责端到端的数据传输，保障传输质量

  • TCP协议：可靠传输协议三次握手，数据完整不丢失，用于网页、文件传输：通过三次握手建立连接，数据不丢失、不乱序、完整性高，适用于文件传输、网页浏览等严谨场景☆收藏
  • UDP协议：快速传输协议无连接，用于直播、网络游戏等实时场景：无需握手、无连接确认，传输速度极快，不保证数据百分百送达，适用于直播、网络游戏、视频通话等实时场景☆收藏

3. 网络层：核心协议为IP协议，负责设备寻址、规划数据传输路由路径，确定数据从发送端到接收端的传输路线☆收藏

4. 网络接口层：最底层，负责将数字数据转换为光电信号，通过网线、光纤、Wi-Fi等物理介质完成数据传输

四、TCP三次握手（完整流程+核心目的 必考）

1. 第一次握手（请求连接）：客户端（电脑/手机）向服务器发送SYN请求，告知服务器“请求建立通信连接”

2. 第二次握手（应答确认）：服务器接收请求后，返回SYN+ACK报文，告知客户端“已收到请求，可准备连接”

3. 第三次握手（最终确认）：客户端接收应答后，发送ACK确认报文，告知服务器“设备已就绪，正式开始数据传输”

核心目的：双向校验客户端与服务器的收发能力，确保通信链路通畅，避免无效连接占用服务器资源，保障传输可靠性☆收藏

💡三次握手是TCP协议建立连接的标准流程，考试简答、填空高频考点

五、数据封装与解封装完整过程（必考流程）

1. 发送端-数据封装：原始数据 → 传输层添加端口信息 → 网络层添加IP地址信息 → 网络接口层添加物理地址信息 → 转换为光电信号发出

2. 接收端-数据解封装：接收光电信号 → 逐层剥离各层协议报头 → 层层校验解析 → 还原为原始数据

2. ping测试网络连通性与延迟：测试本机与目标设备/服务器的网络连通性，判断网络是否通畅、延迟高低☆收藏

3. tracert追踪数据包传输路径，排查网络故障：追踪数据包传输全程路径，显示数据经过的所有路由器节点，用于排查网络卡顿、中断问题☆收藏

七、电子邮件传输原理与协议区别（高频考点）

1. 邮件传输流程：用户客户端 → SMTP发件服务器 → 互联网传输 → 收件服务器 → POP3/IMAP协议推送至用户客户端

2. POP3与IMAP核心区别：

POP3：仅将邮件下载至本地设备，多设备无法同步邮件读取状态

IMAP：支持云端同步，多设备可实时同步邮件已读、未读、删除状态，适配多终端使用

二、网络层安全风险（冷门攻击考点全覆盖）

1. 中间人攻击拦截、窃取、篡改传输数据：拦截、窃取、篡改设备传输数据。 ☆收藏

2. WiFi钓鱼伪造虚假WiFi诱骗设备连接：伪造虚假WiFi，诱导设备连接，窃取数据。 ☆收藏

3. DDoS攻击海量请求挤爆服务器致瘫痪：海量垃圾请求挤爆服务器，导致系统瘫痪、无法使用。 ☆收藏

4. 数据丢包传输过程数据丢失：数据在传输途中出现丢失现象。 ☆收藏

5. 窃听非法截取传输信息：非法监听、截取设备之间的通信内容。 ☆收藏

6. 非法入侵外部设备强行接入网络：陌生设备非法接入物联网内部网络。 ☆收藏

💡幽默解读：网络层是数据传输的“公路”，路上会遇到偷数据、堵路、偷听等各类安全威胁，必须做好防护。

三、应用层安全风险（平台与权限类高频考点）

1. 用户隐私数据泄露个人、设备数据外泄：用户隐私数据、日志数据泄露。 ☆收藏

2. 平台组件漏洞软件平台自身安全缺陷：平台组件存在安全漏洞、API接口滥用。 ☆收藏

3. 权限管理混乱普通用户越权操作，必考考点：权限管理混乱，普通用户非法获取管理员权限。 ☆收藏

💡幽默解读：应用层是整个系统的“指挥大厅”，一旦权限、数据出问题，整个管理体系都会失控。

四、全维度防护措施（简答标准答案，必背）

1. 设备防护

加固硬件、定期检修、防水防破坏。 ☆收藏

2. 网络防护

数据加密、WiFi密码加固、关闭陌生端口。 ☆收藏

3. 身份防护

实体认证+消息认证、多因素登录、权限分级。 ☆收藏

4. 系统防护

定期更新固件、修复漏洞、备份数据。 ☆收藏

💡幽默解读：防护要从硬件、网络、身份、系统全方位下手，层层把关才能抵御各类攻击。

第二节 物联网虚拟身份（全新必考知识点）

虚拟身份定义网络中唯一数字标识，名词解释：物联网场景中，设备和用户在网络中的唯一数字标识，无实体形态，用于身份校验。 ☆收藏

一、两类虚拟身份（区分考点）

1. 用户虚拟身份使用者网络身份：账号密码、人脸、指纹、登录令牌。 ☆收藏

2. 设备虚拟身份联网设备专属编号：设备ID、MAC地址、SN序列号、唯一设备编码。 ☆收藏

💡幽默解读：虚拟身份就像网络世界的身份证，人和设备都有专属“证件”，没有合法身份就无法联网接入。

第三节 物联网自主可控技术（单元压轴大题、简答核心考点）

自主可控完整定义全自研、不依赖外部技术：核心技术、硬件设备、系统软件、研发维护全部由国内自主研发、自主生产、自主升级、自主维护，不依赖国外技术，无外部后门风险。 ☆收藏

核心意义（四点必背，考试高频简答）

  • 1. 保障国家网络安全、信息安全。 ☆收藏
  • 2. 打破国外技术垄断，不被“卡脖子”。 ☆收藏
  • 3. 稳定供应链，保障设备批量应用。 ☆收藏
  • 4. 拥有自主修改、升级、优化权限，适配国内场景。 ☆收藏
💡幽默解读：自主可控就是核心技术握在自己手里，自己说了算，不用受制于他人，安全和发展都有保障。
