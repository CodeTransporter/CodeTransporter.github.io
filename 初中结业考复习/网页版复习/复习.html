第一单元 互联网及其影响（互联网博物馆策展）

一、互联网核心基础概念（必考）

1. 互联网完整定义：互联网又称因特网，是全球规模最大、开放程度最高的信息网络体系，由无数个独立的计算机网络通过路由器等网络设备相互连接，实现全网互联互通、信息资源共享的巨型网络系统。
2. 主机定义：所有能够接入互联网的终端设备（计算机、手机、平板、智能家电等），在网络体系中统一被称为主机，是网络数据收发的基本单元。

3. 阿帕网（ARPANET）完整知识点

诞生背景：20世纪60年代，包交换技术取得技术突破，为阿帕网搭建奠定核心技术基础。
关键事件：1969年10月，美国加州大学洛杉矶分校与斯坦福研究所的计算机，通过阿帕网完成首次跨节点远程通信。
核心意义：本次通信虽仅完成部分数据传输，但标志着互联网雏形正式诞生，证明跨地域计算机可通过网络实现数据交换，打破空间通信壁垒。

二、互联网完整发展历程（全时间节点）

  • 1. 20世纪60年代：阿帕网诞生，互联网雏形出现
  • 2. 20世纪70年代：TCP/IP协议正式诞生，统一全网通信标准，实现不同网络、不同设备之间的互联互通
  • 3. 20世纪90年代：万维网（WWW）问世并商业化，简化互联网使用门槛，推动互联网从专业领域走向大众普及
  • 4. 21世纪初：智能手机普及，移动互联网时代来临，全面改变大众生产生活方式
  • 5. 21世纪中后期：5G技术全面商用，全球进入万物互联时代；生成式人工智能兴起，开启互联网智能化交互新阶段

三、IP地址核心知识点（重难点·全解析）

1. IP地址定义：IP地址是互联网中每一台主机的唯一逻辑标识，相当于设备在网络中的专属“住址”，是网络识别、定位设备的核心依据。
2. IP地址完整结构：由网络号 + 主机号两部分组成，二者缺一不可
网络号：标识设备所属的网络区域（类比邮政编码，确定大范围区域）
主机号：标识该网络区域内的具体设备（类比门牌号，确定精准设备）

3. IPv4 完整概念

组成：采用32位二进制数编制，分为4组8位二进制数，转换为十进制展示（例：192.168.1.1）
特点：地址总量固定、数量有限，无法适配当下海量物联网设备接入需求，目前已面临地址枯竭问题

4. IPv6 完整概念

组成：采用128位二进制数编制
特点：地址空间呈指数级扩大，可满足全球万物互联的设备接入需求，具备安全性高、传输稳定、适配性强的优势，是未来互联网的核心地址协议

5. IPv4与IPv6核心对比（必背）

IPv4：32位二进制、地址有限、安全性较弱、传统互联网通用
IPv6：128位二进制、地址无限、安全性更强、适配物联网/智能设备

四、互联网应用及完整特征

1. 常见互联网应用分类

信息检索类：各类搜索引擎；社交沟通类：即时通讯软件；学习办公类：在线课堂、云端办公平台；生活服务类：网购、外卖、在线支付；娱乐休闲类：视频、游戏平台

2. 互联网应用完整核心特征

  • ① 无时空限制：可随时随地联网使用，打破地域、时间束缚
  • ② 非接触式交互：全程线上数据交互，无需线下接触即可完成服务
  • ③ 服务个性化、多元化：依托算法推送，适配不同用户需求，服务场景全覆盖
  • ④ 呈现形式多媒体化：从早期单一文本，升级为图文、音频、视频融合呈现
  • ⑤ 功能集成融合化：单一应用可集成多项功能，告别单一服务模式，提升使用效率

五、互联网双重影响（辩证考点·完整版）

1. 互联网创新积极价值

  • ① 信息领域：打破信息壁垒，信息检索、传播高效便捷，实现全球信息共享
  • ② 经济领域：催生电商、共享经济、数字支付等新业态，“互联网+”赋能传统农业、工业、服务业，重构产业生态、优化资源配置
  • ③ 教育领域：打破教育资源地域差异，在线教育、远程研学实现优质资源普惠共享
  • ④ 社会领域：电子政务、智慧城市、在线医疗提升公共服务效率，助力文化跨地域传播交流
2015年国家正式提出“互联网+”行动计划，推动互联网与实体经济深度融合。

2. 互联网潜在消极风险（全考点）

  • ① 信息过载与虚假信息：网络信息繁杂冗余，不实谣言、伪科学内容快速扩散，误导公众认知
  • ② 网络暴力与不良文化：存在言语攻击、人肉搜索、网络欺凌等行为，低俗内容侵蚀网络风气
  • ③ 青少年网络成瘾：过度沉迷网络游戏、短视频，导致学业荒废、社交能力退化、心理健康受损
  • ④ 数字鸿沟：老年人、偏远地区、贫困群体因设备缺失、数字技能不足，无法平等享受网络服务，加剧社会资源分配不均
  • ⑤ 隐私泄露与网络诈骗：平台过度授权导致个人信息（手机号、住址、消费记录）泄露，不法分子利用泄露信息实施精准诈骗、骚扰

第二单元 中华优秀传统文化数字传播（互联网信息处理）

一、五大信息获取方式（完整对比考点）

1. 查阅纸质资料：优势是内容权威、无网络虚假信息；不足是更新慢、携带不便、检索效率低
2. 询问他人：优势是获取信息快速、针对性强；不足是受个人认知局限，信息主观性强、易出错
3. 实地考察：优势是获取一手真实资料、体验感强；不足是耗时耗力、成本高、受地域限制
4. 互联网搜索：优势是信息量大、检索快速、不受时空限制；不足是信息参差不齐、需人工甄别真伪
5. 生成式AI工具：优势是可自主生成、整合信息，效率极高；不足是存在生成错误信息风险，需核验权威来源

二、搜索引擎核心知识点（重难点）

1. 搜索引擎定义：依托爬虫程序、索引数据库，帮助用户在海量网络信息中快速检索、匹配所需资源的网络工具，可检索网页、图片、视频、资讯等各类内容。
2. 搜索引擎完整工作三步法（必考）
  • ① 抓取：通过网络爬虫（蜘蛛程序）自动遍历全网网页，采集各类信息资源（爬虫属于正规工具，并非病毒）
  • ② 索引：对抓取的海量数据进行分类、整理、标注关键词，搭建专属索引数据库，实现快速检索储备
  • ③ 排序：依托算法，根据用户关键词匹配度、网页权威性、内容相关性，对检索结果优先级排序，优质内容前置展示
3. 搜索引擎四大分类：全文搜索引擎、垂直搜索引擎（细分领域专属）、元搜索引擎（整合多平台结果）、目录搜索引擎（人工分类整理）

三、高阶搜索技巧（实操+考点）

1. 精确匹配技巧：使用英文双引号""包裹关键词，禁止系统拆分词语，实现精准检索（例："广东醒狮文化"）
2. 关键词优化：摒弃口语化语句，使用简洁、专业的词组作为关键词，可多关键词空格组合缩小范围
3. 分类筛选：检索时可精准选择网页、图片、视频、资讯等类型，过滤无效资源
4. 权威资源优先：优先使用数字故宫、数字敦煌、故宫名画记、数字多宝阁、数字藏经洞等官方文化平台资源，内容权威无版权问题

四、网络素材下载规范与格式

1. 文字素材：选中复制粘贴，保存为TXT、WPS、Word格式
2. 图片素材：右键“图像另存为”，常用格式JPG、JPEG、PNG、GIF、BMP；模糊图片可使用识图工具以图搜原图
3. 音视频素材：正规平台下载，音频常用MP3、WMA、WAV；视频常用MP4、AVI、FLV；格式不兼容可使用格式转换工具处理

五、互联网信息管理核心知识

1. 信息分类依据：可按照文件类型、主题内容、创建时间、文件大小四大维度分类整理网络资源
2. 目录树结构定义：以根目录为起点，逐层拆分一级、二级子文件夹，形成倒立树形存储结构，层级清晰，方便文件快速查找、归类、管理，是计算机资源管理的核心方式
3. 两种信息存储管理方式（完整对比）
本地存储管理（硬盘、U盘、移动硬盘）
优势：无需网络即可读写、数据隐私性强、无第三方泄露风险、读写稳定
不足：物理介质易损坏、丢失、老化，无法跨设备同步，远程调用不便
云端在线管理（网盘、云文档、在线协作平台）
优势：支持多设备实时同步、多人在线协作、远程随时访问、无需占用本地硬件空间
不足：完全依赖网络网速，数据存储于第三方服务器，存在隐私泄露、平台关停丢失数据风险

第三单元 利用互联网推广中国非遗文化（网站制作）

一、网页与网站基础数据构成

一个完整的网页/网站包含五类核心数据：文本数据、图像数据、音频数据、视频数据、用户行为数据
用户行为数据：用户浏览网页时的点击记录、停留时长、页面跳转、搜索记录等后台统计数据

二、网站导航系统完整分类

1. 全局导航：固定布置在网页顶部，覆盖全站所有页面，可实现任意子页面一键跳转返回主页，是网站核心导航
2. 局部导航：仅适配当前栏目、当前页面的专属导航，服务于细分内容浏览
3. 面包屑导航：层级式导航（例：主页>传统技艺>龙泉青瓷），清晰展示用户当前浏览位置，方便用户回溯上级页面，提升浏览体验

三、HTML核心标签大全（定义+属性+用法·必考）

1. 网页基础骨架标签（层级固定）

<html>：网页根标签，包裹网页所有代码内容，是整个网页的最外层结构
<head>：网页头部标签，存放网页元数据、标题、样式配置，内容不会在页面可视化展示
<title>：网页标题标签，定义浏览器标题栏显示的文字名称
<body>：网页主体标签，所有可视化的文本、图片、视频、按钮等内容均存放于此

2. 文本排版标签

<h1>~<h6>：六级标题标签，h1字号最大、权重最高，h6最小，用于页面标题层级区分
<p>：段落标签，自动实现段落换行、预留间距，用于正文文本排版

3. 媒体与超链接标签（核心属性）

<img>：图片单标签，用于插入网页图片
核心属性：src（必填，图片存储路径/网址）、alt（图片加载失败时的替代文字）、width/height（自定义图片宽高，单位px）
<a>：超链接双标签，实现页面跳转
核心属性：href（必填，跳转目标网址/页面路径），格式：<a href="目标地址">显示文字</a>

4. 表格标签

<table>：表格外框标签，定义整体表格
<tr>：表格行标签，定义表格中的一行内容
<td>：单元格标签，定义表格中单个单元格，存放具体内容

四、HTML标签核心规则（必背考点）

1. 双标签：成对出现，有开始标签和结束标签（例：<p></p>、<body></body>）
2. 单标签：无结束标签，独立使用（例：<img>、<br>换行标签）
3. 固定层级关系：<html>包含<head>和<body>，<head>与<body>为并列关系，不可嵌套错乱

五、网页美化与交互技术

1. CSS样式：用于网页静态美化，核心属性：background-color（背景颜色）、background-image（背景图片）、opacity（透明度，数值0-1，0完全透明、1完全不透明）
2. JavaScript（JS）：用于实现网页动态交互效果，让网页具备点击响应、动态切换等互动功能

六、网站完整发布流程（两种方式全解析）

1. 局域网发布（校内/内网使用）

依托Windows系统自带IIS服务器搭建，配置完成后，仅同一局域网内的设备可通过IP地址访问网站，适合校园内部展示、测试使用

2. 互联网公网发布（全网可访问）四步流程

① 申请合法域名 ② 购买/托管云端服务器 ③ 完成ICP备案（国内网站必备） ④ DNS解析绑定域名与IP

七、域名与DNS解析重难点

1. 常见域名后缀含义
地区类：cn（中国）、hk（中国香港）、us（美国）
机构类：gov（政府机构）、edu（教育机构）、com（商业机构）、net（网络服务机构）、org（非营利组织）
2. DNS解析完整定义：DNS即域名系统，相当于网络电话簿，核心作用是将人类易记忆的域名，自动转换为设备可识别的IP地址，解决IP地址晦涩难记的问题，是网页访问的核心前置步骤。

第四单元 搭建校园网络系统（互联网基本原理）

一、网络连接方式与设备考点

1. 有线传输方式

双绞线：短距离传输、成本低、适用性广，多用于校园教室、办公室局域网布线
光纤：传输带宽大、速度快、抗干扰能力极强、传输距离远，是骨干网络核心传输介质

2. 无线传输方式

Wi‑Fi：覆盖范围广、传输速率高，适合室内大面积设备联网
蓝牙：传输距离近、功耗极低，适合手机、耳机、外设等短距离设备互联

二、网络类型完整定义

1. LAN局域网：覆盖范围小（单教室、办公室、校园）、传输速度快、延迟低、搭建成本低，仅限局部区域设备互联
2. WAN广域网：覆盖范围极大（跨城市、跨省、跨国），传输距离远，互联网本质是无数局域网通过广域网交织形成的巨型网络
3. 路由器核心作用：唯一可实现局域网与广域网互联互通的核心网络设备，负责数据转发、路由寻址

三、TCP/IP四层协议模型（必考重难点·逐层解析）

1. 应用层：面向用户的功能应用层，承载各类网络服务，对应核心协议：
HTTP（网页浏览协议）、SMTP（发送邮件协议）、POP3/IMAP（接收邮件协议）
2. 传输层：负责端到端的数据传输，保障传输质量
TCP协议：可靠传输协议，通过三次握手建立连接，数据不丢失、不乱序、完整性高，适用于文件传输、网页浏览等严谨场景
UDP协议：快速传输协议，无需握手、无连接确认，传输速度极快，不保证数据百分百送达，适用于直播、网络游戏、视频通话等实时场景
3. 网络层：核心协议为IP协议，负责设备寻址、规划数据传输路由路径，确定数据从发送端到接收端的传输路线
4. 网络接口层：最底层，负责将数字数据转换为光电信号，通过网线、光纤、Wi‑Fi等物理介质完成数据传输

四、TCP三次握手（完整流程+核心目的）

1. 第一次握手（请求连接）：客户端（电脑/手机）向服务器发送SYN请求，告知服务器“请求建立通信连接”
2. 第二次握手（应答确认）：服务器接收请求后，返回SYN+ACK报文，告知客户端“已收到请求，可准备连接”
3. 第三次握手（最终确认）：客户端接收应答后，发送ACK确认报文，告知服务器“设备已就绪，正式开始数据传输”
核心目的：双向校验客户端与服务器的收发能力，确保通信链路通畅，避免无效连接占用服务器资源，保障传输可靠性

五、数据封装与解封装完整过程

1. 发送端‑数据封装：原始数据 → 传输层添加端口信息 → 网络层添加IP地址信息 → 网络接口层添加物理地址信息 → 转换为光电信号发出
2. 接收端‑数据解封装：接收光电信号 → 逐层剥离各层协议报头 → 层层校验解析 → 还原为原始数据

六、网络诊断命令（实操必考·完整功能）

1. ipconfig /all：查看本机完整网络配置，包含IP地址、MAC物理地址、子网掩码、网关、DNS服务器地址
2. ping：测试本机与目标设备/服务器的网络连通性，判断网络是否通畅、延迟高低
3. tracert：追踪数据包传输全程路径，显示数据经过的所有路由器节点，用于排查网络卡顿、中断问题

七、电子邮件传输原理与协议区别

1. 邮件传输流程：用户客户端 → SMTP发件服务器 → 互联网传输 → 收件服务器 → POP3/IMAP协议推送至用户客户端
2. POP3与IMAP核心区别
POP3：仅将邮件下载至本地设备，多设备无法同步邮件读取状态
IMAP：支持云端同步，多设备可实时同步邮件已读、未读、删除状态，适配多终端使用

四、守护数据之钥——互联网信息加密（核心重难点）

1. 加密技术四大基础概念（教材定义）
明文：未经加密、可以直接阅读、理解的原始数据信息
密文：经过加密算法处理后，无规律、无法直接阅读和解读的加密数据
密钥：控制加密、解密过程的专属关键参数，密钥不同，加密结果不同
加密：将明文通过算法转换为密文的过程，用于保护数据安全
解密：持有合法密钥，将密文还原为原始明文的过程
经典案例：凯撒密码（基础对称加密），通过固定字母位移实现简单加密解密。
2. 两大加密方式完整对比（必考）
对称加密：加密、解密全程使用同一组密钥，运算速度快、效率高、适合大批量文件加密；缺点是密钥传输、存储存在泄露风险。
非对称加密：采用公钥、私钥成对密钥，公钥公开用于加密，私钥私密保存用于解密；优点是安全性极高、不易破解；缺点是运算速度慢、耗时更长。
3. 日常文件三种加密实操方式
  • ① 文档加密：WPS/Word设置打开密码、编辑权限密码，限制他人查看、修改
  • ② 压缩包加密：WinRAR、7-Zip工具对压缩文件设置密码，保护批量文件
  • ③ 磁盘加密：电脑BitLocker磁盘加密，保护整盘数据不被窃取
4. 前沿加密技术拓展
量子加密（绝对安全、无法窃听破解）、区块链加密（数据不可篡改、全程可追溯）、AI智能加密防御（实时识别攻击、自动防护）

第一单元 校园“物联网”改造——初识物联网

1. 物联网完整官方定义（教材原话必考）

通过传感器、二维码识读设备、射频识别（RFID）、红外感应器、全球定位系统、激光扫描器等信息传感设备，按约定的通信协议，将任意物品与互联网连接起来，进行信息交换和通信，最终实现物品的智能化识别、定位、跟踪、监控和管理的新型网络架构。
幽默注解：以前的互联网是“人上网聊天查资料”，物联网是“万物开口说话、主动上报状态”，让普通家电、设备、环境都变成会汇报、能自控的“智能打工人”。

2. 物联网中“物”的严格限定（易漏考点）

并不是所有东西都能叫物联网终端！必须是经过智能化改造、搭载传感/通信模块、可联网交互的物品，普通桌椅、风扇、水杯不算物联网设备。
幽默注解：普通物品是“哑巴”，物联网物品是“会联网说话的聪明人”，没改装的普通物件根本没资格进物联网圈子。

3. 物联网四大核心特性（补全细碎考点）

  • ① 全面感知：全天候、全方位采集物理世界数据，不遗漏环境、设备细微变化
  • ② 可靠传递：数据传输稳定、低丢失、低错乱，弱网络环境也能适配基础传输
  • ③ 智能处理：依托大数据、AI分析数据，不靠人工主观判断，自动输出决策
  • ④ 双向交互（超易漏考点）：不止设备上传数据，平台可以下发指令控制设备，是双向沟通，不是单向汇报
幽默注解：感知是“眼睛看”，传递是“嘴巴说”，智能处理是“大脑想”，双向交互是“手脚动”，一套完整智能操作，不是只会看不会动的摆设。

4. 物联网发展完整时间线（补齐所有冷门事件）

  • ① 1982年：卡内基梅隆大学 联网可乐售卖机（全球首个物联网设备），工作人员联网就能看有没有可乐、是否冷藏正常
  • ② 1991年：剑桥大学 特洛伊咖啡壶（初代物联网可视化应用，解决“白跑一趟看咖啡”的麻烦）
  • ③ 1995年：比尔盖茨《未来之路》首次提出万物互联构想，物联网概念雏形诞生
  • ④ 1999年：美国MIT正式定义“物联网”，核心依托RFID射频识别技术落地
  • ⑤ 1999年：中国启动物联网相关技术研究
  • ⑥ 2009年：物联网纳入中国国家战略产业
  • ⑦ 2010年：物联网首次写入中国政府工作报告，迎来高速发展期
幽默注解：物联网不是突然冒出来的黑科技，是从“看可乐、看咖啡”的小工具，一步步进化成现在的全屋智能、智慧城市。

5. 互联网VS物联网 超详细对比（补齐区别细节）

✅ 相同点：均依托互联网基础网络架构，遵循网络通信协议，实现数据传输交换
❌ 核心不同点（考试高频）：
  • ① 连接对象：互联网=人、电脑；物联网=物、人、设备
  • ② 核心功能：互联网=信息浏览、社交、资源共享；物联网=感知、监测、自动控制、智能管理
  • ③ 工作模式：互联网=人工主动操作；物联网=全自动、无人值守为主
  • ④ 数据特点：互联网=大数据、碎片化娱乐/学习数据；物联网=小数据、高频次、实时性传感数据
幽默注解：互联网是“人类社交工具”，物联网是“设备管家工具”，一个服务人娱乐学习，一个帮人干活值守。

6. 物联网三层架构 逐层超全解析（补齐所有层功能、设备、易错点）

（1）感知层——物联网的“五官皮肤”

核心功能：唯一负责物理信号转数字信号，是物联网数据的唯一来源，没有感知层就没有物联网数据。
完整设备清单：各类传感器（温湿度、光照、红外、噪声）、RFID标签、二维码、GPS定位、摄像头、生物识别设备
易错点：感知层只采集数据，不传输、不分析、不决策

（2）网络层——物联网的“神经血管”

核心功能：承担数据上传、指令下发双向传输任务，打通感知层和应用层的通道。
完整传输介质&技术：有线（以太网、光纤、同轴电缆）、无线（Wi-Fi、蓝牙、ZigBee、LoRa、NB-IoT、5G）
核心协议：IPv6、MQTT、TCP、UDP、HTTP
易错点：网络层只传数据，不采集、不分析数据

（3）应用层——物联网的“大脑中枢”

核心功能：处理、分析、存储数据，输出智能决策、执行控制、展示服务。
核心技术：云计算、大数据分析、人工智能、平台软件、安全加密、可视化界面
常见应用场景：后台管理平台、手机控制端、智能预警系统、自动调控系统
幽默注解：感知层负责“摸情况”，网络层负责“传消息”，应用层负责“做决定、下命令”，三层分工明确，缺一必瘫痪。

7. 物联网与普通物联系统 核心区别（超易漏考点）

① 物联网：宏观、通用的技术体系，是一整套互联规则和技术合集
② 物联系统：微观、具体的落地产品，由硬件+网络+软件组成，比如智慧校园监测系统、景区管理系统
幽默注解：物联网是“整套行业规则”，物联系统是“按照规则做出来的成品工具”。

8. 物联网全场景应用细分（补齐冷门应用）

除四大主流场景，补齐教材冷门考点：智慧仓储、智能安防、智慧环保、远程设备运维

第二单元 景区智能管理系统——感知层初体验

1. 二维码技术 全套超细知识点（补齐所有遗漏细节）

① 编码原理：黑白点阵对应二进制0和1，通过点阵排列存储海量信息
② 完整结构（必考填空）：定位图形、校正图形、数据编码区、纠错冗余区
③ 纠错能力（超易漏）：自带冗余纠错，破损30%以内仍可正常识别
④ 完整识别流程：光学扫描→光电信号转换→译码解析→信息输出展示
⑤ 全部安全隐患：伪造仿冒二维码、恶意跳转钓鱼链接、隐私信息裸漏、被他人私自扫码窃取信息
⑥ 全套防护手段：动态刷新二维码、内容加密、添加专属防伪水印、限制扫码权限、官方专用扫码工具、定期更换二维码
幽默注解：二维码看着简单，实则自带“容错buff”，破一点没事，但最怕被人偷偷换假码，妥妥的“双面小工具”。

2. RFID射频识别 补齐所有冷门考点

① 工作原理：读卡器发射无线电波→激活电子标签→标签返回存储数据→读卡器接收解析→系统处理
② 电子标签分类（超易漏）：有源标签（自带电池、传输远）、无源标签（无电池、靠射频供电、成本低、最常用）
③ 完整优缺点：除基础优势外，补充——无视觉遮挡要求、可穿透非金属物体识别；致命短板：金属、液体完全屏蔽信号，考场、景区设备需规避
④ 核心应用细节：景区电子围栏预警、游客轨迹追踪、门票防伪核验、设备资产管理
幽默注解：RFID就是“隐形身份卡”，不用扫码不用碰，刷一下就识别，唯独怕水和金属，堪称“娇气的高效工具”。

3. 红外传感器 100%补齐遗漏原理+实操规则

① 核心工作原理：发射器发射红外光束，接收器接收光束，通过光束遮挡/反射状态判断物体有无
② 两类传感器完整对比（补齐细节）：
- 对射式：收发分离、光束集中、抗干扰极强、检测距离远（数米至数十米），适合精准计数、安防预警
- 反射式：收发一体、靠物体反射光线工作、距离近、易受光线干扰，适合近距离感应、自动设备
③ 人流量统计完整逻辑（教材原版）：双红外配对使用，通过遮挡顺序判断进出，软件算法防抖、防重复计数、防误触发
幽默注解：对射式是“远距离哨兵”，眼神好、抗干扰；反射式是“近距离小助理”，方便但容易受环境影响。

4. 多传感器融合技术 补齐所有流程+核心优势

① 核心定义：单一传感器数据误差大、信息片面，通过算法整合多类传感器数据，弥补单一设备缺陷
② 完整五步流程（一字不差教材版）：数据采集→数据预处理（去噪、校准、筛选无效数据）→特征提取→多源数据融合计算→智能决策与设备控制
③ 核心优势细化：降低环境干扰、提升数据精准度、覆盖多维监测场景、设备容错率大幅提升，单个传感器故障不影响整体工作
④ 景区全套监测传感器（补齐全部）：温湿度、光照强度、PM2.5、CO₂浓度、环境噪声、二氧化硫、氮氧化物、水质传感器
幽默注解：一个传感器容易“看走眼”，多个传感器抱团工作，互相纠错，数据精准度直接拉满，堪称物联网的“团队协作天花板”。

第三单元 智慧校园系统——网络层初体验

1. 数据通信完整分类（补齐遗漏判定标准）

① 按传输介质：有线通信、无线通信
② 按传输距离：短距离通信、长距离广域通信
③ 按传输信号：数字通信（物联网主流、抗干扰、精准）、模拟通信（老旧设备使用）
④ 按通信方式：一对一单播、一对多组播、全网广播
幽默注解：通信方式分很多种，就像聊天，单独私聊、小组群聊、全班广播，各司其职。

2. 六大无线通信技术 【完整定义+特点+优缺点+适用场景】（彻底补齐，学生彻底看懂）

（1）Wi-Fi 无线局域网

定义：基于WLAN标准的短中距离高速无线通信技术，将有线网络信号转化为无线信号，支持多设备并发联网。
频段详解：2.4GHz（穿墙能力强、覆盖广、干扰多、速度一般）；5GHz（速率高、延迟低、干扰少、穿墙弱）
优缺点：高速传输、适配大流量数据；功耗偏高、远距离信号衰减快
适用场景：校园上网、视频传输、大屏设备、大数据同步
幽默注解：Wi-Fi就是网络界的“大力士”，传大文件、看视频一绝，就是有点费电、远了就没信号。

（2）蓝牙 Bluetooth

定义：超短距离、低功耗的个人局域网通信技术，主打近距离小型设备配对交互。
核心参数：有效距离10米内，低功耗、小数据传输、双向通信
优缺点：省电、配对简单、延迟低；传输速度慢、距离极短
适用场景：手环、耳机、智能灯具、小型传感器指令传输
幽默注解：蓝牙是“贴身小助理”，只跟身边设备互动，省电好用，就是不能远距离干活。

（3）ZigBee 紫蜂协议

定义：专为物联网设计的低功耗、低速率、大容量自组网无线通信技术。
核心特点：功耗极低、可容纳上百个设备组网、自动组网、自动修复网络链路
优缺点：超级省电、适合长期待机；传输速度慢，不适合大文件
适用场景：校园传感器集群、全屋智能设备、长期电池供电设备
幽默注解：ZigBee是物联网的“省电劳模”，一次用电待机半年，就是干活速度慢，只适合传小数据。

（4）LoRa 远距离低功耗通信

定义：远距离、低功耗、强穿墙的广域物联网无线通信技术。
核心特点：传输距离可达3-5公里、抗干扰强、穿墙能力优秀、超低功耗
优缺点：距离远、信号稳；传输速率极低
适用场景：校园大范围环境监测、园区安防、野外传感设备
幽默注解：LoRa是“长跑选手”，跑得远、耐造、省电，就是干活效率低，不适合高速传输。

（5）NB-IoT 窄带物联网

定义：依托运营商蜂窝基站的广覆盖、超低功耗物联网专用通信技术。
核心特点：全网覆盖、无需自建基站、功耗极低、支持海量设备接入
优缺点：覆盖无死角、不用布线组网；速率低、依赖运营商信号
适用场景：智能水表、气表、路灯、智慧城市远程监测
幽默注解：NB-IoT是“全域网格员”，哪里有手机信号哪里就能用，不用自己搭设备，省心又省电。

（6）5G 第五代移动通信

定义：新一代蜂窝移动通信技术，是物联网高速实时交互的核心技术。

四、两类温度传感器 教材全细节

1. 热电偶
核心原理：热电效应（跨学科物理考点），两种不同金属/半导体导体组成闭合回路，两端温差产生电势差，以此换算温度。

幽默注解：测温界的“硬汉”，不怕高温、不怕造，工业场景专属劳模。
2. 热敏电阻
核心原理：电阻阻值随环境温度变化发生规律性改变，控制器通过电阻变化换算温度数值。
分类：金属热电阻、半导体热电阻
核心优点：体积超小、灵敏度极高、响应速度快、适配常温环境
适用场景：家用电器、智能设备、室内环境监测
幽默注解：测温界的“小机灵鬼”，小巧灵敏，家用设备全覆盖。

    time.sleep(0.5) # 绿灯亮0.5秒

from unihiker_k10 import wifi, mqttclient, servo
import time

# 初始化舵机，绑定P0接口

wifi.connect(ssid="WiFi名",psd="WiFi密码",timeout=50000)
mqttclient.connect(server="192.168.43.86",port=1883,user="siot",psd="dfrobot")

# 接收舵机控制消息
def servo_recv(msg):
    try:
        angle = int(msg) # 接收的字符串角度转为数字
        angle = max(0,min(180,angle)) # 强制限制角度0-180，保护设备
        s1.angle(angle) # 舵机旋转至指定角度
        print("已执行角度：",angle)
    except Exception as e:
        print("指令无效",e)

# 订阅舵机控制主题
mqttclient.subscribe("siot/servo",servo_recv)

# 持续监听指令
while True:

第六单元 物联网安全保卫战——物联网安全与发展

核心总纲：物联网三层安全体系（感知层+网络层+应用层）

一、感知层安全风险（教材100%全覆盖，补齐传感器欺骗考点）

1. 物理风险：设备被人为破坏、拆卸、浸水、磕碰损坏
2. 硬件缺陷：设备出厂漏洞、传感器精度故障、硬件老化
3. 软件风险：设备固件漏洞、植入恶意程序
4. 数据欺骗（超重点）：人为伪造环境数据，如热水袋伪造体温、遮挡传感器伪造正常环境，导致系统误判
幽默注解：感知层是物联网的“眼睛皮肤”，眼睛被挡、皮肤被骗，整个系统都会出错。

二、网络层安全风险（补齐所有冷门攻击考点）

1. 中间人攻击：拦截、窃取、篡改设备传输数据
2. WiFi钓鱼：伪造虚假WiFi，诱导设备连接，窃取数据
3. DDoS攻击：海量垃圾请求挤爆服务器，导致系统瘫痪、无法使用
4. 数据丢包、窃听、非法入侵

三、应用层安全风险

1. 用户隐私数据泄露、日志数据泄露
2. 平台组件漏洞、API接口滥用
3. 权限管理混乱，普通用户非法获取管理员权限

四、全维度防护措施（教材标准答案）

1. 设备防护：加固硬件、定期检修、防水防破坏
2. 网络防护：数据加密、WiFi密码加固、关闭陌生端口
3. 身份防护：实体认证+消息认证、多因素登录、权限分级
4. 系统防护：定期更新固件、修复漏洞、备份数据

第二节 物联网虚拟身份（必考全新知识点）

一、虚拟身份定义

物联网场景中，设备和用户在网络中的唯一数字标识，无实体形态，用于身份校验。

二、两类虚拟身份

1. 用户虚拟身份：账号密码、人脸、指纹、登录令牌
2. 设备虚拟身份：设备ID、MAC地址、SN序列号、唯一设备编码
幽默注解：虚拟身份就是网络世界的“身份证号”，没有合法身份，一律不准联网接入。
